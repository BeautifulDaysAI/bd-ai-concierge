## ☀️ 朝のチェック係（毎日 7:00-7:30）

**ローテーション制：4名で週1回ずつ**

### 手順

1. Slack で「☀️ 朝の市況ドラフトが準備できました」通知を確認
2. `/admin/morning-brief` を開く
3. AI が生成したドラフトを確認
4. **チェック項目**：
   - [ ] 数値（株価、為替、指標）の正確性
   - [ ] 出典の妥当性
   - [ ] 「絶対」「必ず」「保証」など断定表現がない
   - [ ] 個別商品名（銘柄、保険名）が含まれていない
   - [ ] 「AI作成・担当者確認済」と末尾に明記されている
   - [ ] BD のトーンに合っている
5. 必要に応じて編集
6. 「承認して配信予約」ボタンをクリック
7. **これで完了！** 7:30 に自動配信される

### NG なら却下

明らかに不適切な内容（誤情報、断定表現、商品推奨）が含まれている場合：
- 該当箇所を削除し、代わりに「（本日の市況サマリは見送りです）」とだけ書いて承認
- 新もとに Slack で報告（プロンプト改善のため）

### チェック時間の目安

慣れれば1回 5〜10分。

## 📩 資料お預かり対応（随時）

### 通知が来たら

Slack で「📩 〇〇様から資料を受領しました」が届く

### 手順

1. `/admin/documents` を開く
2. 該当の会員様を確認
3. 担当 FP が次回相談時までに内容を確認
4. 相談後、`/admin/documents` で「確認済」マークをつける（Week 5で実装予定）

---

## 🚨 Lv.3 案件対応（随時）

### 通知が来たら

Slack で「🚨 個別商品名を含む相談がありました」が届く

### 手順

1. `/admin/messages` を開く
2. 該当会員のメッセージを確認
3. **24時間以内に担当 FP からフォロー連絡**
   - LINE で「ご連絡が遅くなりました」と一声かける
   - 担当 FP との相談予約を提案
4. 必要に応じて、その場で予約調整

### 重要

Lv.3案件は「商品判断」「契約判断」を含む重要案件です。
**放置しない・AIに任せきりにしない**ことが信頼の基盤になります。

   - NGワードリスト追加（`src/lib/ai/guards/ng-words.ts`）
   - FAQに想定問答を追加

## 📅 イベント管理

### 新規イベント登録

Supabase ダッシュボード → Table Editor → `events`：

```
name: 「2026年6月 月例パーティ」
description: 詳細...
starts_at: 2026-06-15 19:00:00+09
location: 「都内 ◯◯ホテル」
capacity: 30
member_priority: true（Memberプラン優先案内するなら）
```

### 出欠の確認

`/admin/events` で各イベントの参加者数を表示。
詳細は Supabase の `event_attendances` テーブルで会員別に確認可能。

## 👤 会員管理

### プラン変更

Free → Member への昇格：

Supabase Table Editor → `members`：
- 該当会員のレコードを編集
- `plan` を `member` に変更
- 保存

### 退会処理

通常は LINE のブロックで自動的に `deleted_at` が記録されます。
手動で退会させたい場合は、`deleted_at` に現在時刻を設定。

30日後に自動削除されます。

## 🔐 セキュリティ

### 月次

- [ ] Anthropic API キーのローテーション（月1回推奨）
- [ ] Supabase service_role キーが漏洩していないか確認
- [ ] LINE Channel Access Token の有効期限確認
- [ ] 管理画面アクセスログレビュー

### 緊急時

漏洩・不正アクセス疑い時：

1. **すぐに**：
   - 該当キーをローテーション
   - Vercel のデプロイ一時停止
2. **24時間以内**：
   - 影響範囲の特定
   - 必要なら個人情報保護委員会への報告検討
3. **1週間以内**：
   - 顧問弁護士と相談
   - 再発防止策の実装
