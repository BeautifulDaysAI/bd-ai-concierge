# Week 2 機能ガイド

Week 1の基盤に、Week 2で追加された機能の説明です。

---

## ✨ 追加された機能

### 1. メッセージログのDB保存

会員様との全会話が自動的に `messages` テーブルに保存されます。

- 入力（in）・出力（out）の両方向
- フィルターレベル（lv1/lv2/lv3）
- AI モデル名
- NG検出フラグ

→ 週次レビューや品質改善に活用できます。

### 2. 会員自動登録

LINE で初めて話しかけてきた人を自動的に `members` テーブルに登録します。

- LINE プロフィールから displayName を取得
- デフォルトプランは `free`
- 友だち追加時にもウェルカムメッセージで初期化
- ブロック / 退会時はソフトデリート（30日後に物理削除予定）

### 3. BD独自FAQの参照

会員様の質問に関連する BD の公式情報を、AI が自動的に参照して回答します。

サンプル FAQ（`src/lib/ai/knowledge/faq.ts`）：
- 会社について
- サービス内容
- プラン・料金
- パーティ・イベント
- FP相談
- 一般金融用語（NISA等）

**真武さん監修のもとで、実際のBD情報に差し替えてください。**

### 4. 会話履歴を踏まえた応答

過去10件のやり取りを文脈として AI に渡します。
「さっきの話の続きで...」のような自然な会話が可能に。

### 5. NGワードフィルター強化

- **出力側**：AI応答に「絶対」「保証」などが含まれたら差し戻し
- **入力側**：個別商品名（日本生命・SBI証券など）を検出 → 自動的にLv.3扱い
- **契約意図検出**：「解約」「乗り換え」などを検出

### 6. エラーハンドリング強化

- 環境変数の起動時バリデーション
- API 障害時の安全なフォールバック応答
- LINE returnTokenの2重使用エラー回避

### 7. 管理画面の土台

`/admin` で以下を確認できます：

- ダッシュボード（統計サマリ）
- 会話ログ一覧（NG検出ハイライト付き）
- 会員一覧

URL：
- `http://localhost:3000/admin` ダッシュボード
- `http://localhost:3000/admin/messages` 会話ログ
- `http://localhost:3000/admin/members` 会員一覧

### 8. テストコード

```bash
npm test
```

NGワードフィルターと FAQ検索の動作を自動テスト。
コード変更時のリグレッション検出に。

### A. ローカルテスト

```bash

npm run type-check

# 3. 開発サーバー起動
npm run dev
```

### B. LINE で実際に試す

ngrok 経由で実機テスト。Week 1セットアップガイド参照。

試すべきメッセージ例：

| 入力 | 想定レベル | AI 応答 |
|---|---|---|
| 「NISA って何?」 | Lv.1 | 一般説明 + 免責文言 |
| 「Beautiful Days の料金は?」 | Lv.1 | FAQから取得して回答 |
| 「私はいくら積立すべき?」 | Lv.2 | ヒアリング → FP誘導 |
| 「日本生命の保険、解約すべき?」 | Lv.3 | 即 FP 誘導（商品名検出 + 解約検出）|
| 「絶対儲かる方法は?」 | Lv.1 + NG | 差し戻し → SAFE_FALLBACK |

### C. 管理画面で確認

LINE で何件かやり取りした後：

1. `http://localhost:3000/admin` を開く
2. 統計が更新されているか確認
3. `/admin/messages` で会話ログを確認
4. `/admin/members` で自分が登録されているか確認
